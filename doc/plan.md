# BCS 逆向货币系统实现计划

## 项目概述
根据论文《Bidirectional Currency System (BCS)》中的N货币（Being-Needed Money）设计，实现一个轻量级、离线可运行的区块链货币系统。

## 核心需求
1. 离线分布运行，离线支付后同步
2. 零知识证明支持
3. 轻量级区块链（不挖矿，低算力）
4. 身份认证集成
5. 初始化N货币发放 + 定期补充
6. N货币可流转
7. 大规模模拟测试

## 技术选型
- 语言: Python（快速原型 + 系统实现）+ Rust（性能关键模块可选）
- 区块链: 自定义PoA（Proof of Authority）轻链，无挖矿
- 零知识: ZoKrates / 简洁zk-SNARK模拟
- 网络: libp2p或自定义P2P
- 存储: SQLite/LevelDB本地 + IPFS可选
- 模拟: MiroFish自定义模拟器

## 阶段划分

### Stage 1: 系统架构设计
- 输出: 完整系统架构文档、数据模型、接口定义
- 加载技能: 无（自主设计）

### Stage 2: 核心区块链实现
- 轻量级区块链（无挖矿，PoA共识）
- 区块结构、交易模型、Merkle树
- 本地存储、P2P网络基础
- 输出: 可运行的区块链核心

### Stage 3: 离线支付与同步机制
- 离线交易创建与签名
- 交易池与冲突解决
- 同步协议设计
- 输出: 离线支付模块

### Stage 4: 零知识证明集成
- zk-SNARK交易验证
- 余额保密转账
- 输出: ZK模块

### Stage 5: 身份认证与N货币发放
- DID/身份认证系统
- 初始化N货币发放
- 定期补充机制
- 输出: 身份与货币发放模块

### Stage 6: 客户端与API
- REST API / gRPC
- 命令行客户端
- 钱包功能
- 输出: 完整客户端

### Stage 7: 测试
- 单元测试
- 集成测试
- 输出: 测试报告

### Stage 8: 部署
- Docker容器化
- 多节点部署脚本
- 输出: 部署配置

### Stage 9: MiroFish大规模模拟
- 大规模节点模拟
- 交易压力测试
- 网络分区与恢复测试
- 输出: 模拟结果报告
